# Sber MQTT Bridge — интеграция Home Assistant со Сбером

Кастомная интеграция для Home Assistant, которая пробрасывает ваши устройства из HA в экосистему умного дома Сбера через MQTT. 
После настройки устройства появляются в приложении **Салют**, что позволят управлять ими удаленно через интернет, а также 
через голосового помощника.

## Что умеет

- Пробрасывает выбранные устройства HA в приложение Салют
- Поддерживает **реле** (выключатели, лампы, кнопки, сценарии), **датчики температуры/влажности**, **сценарные кнопки** (прокидывают события из HA в Салют), **кондиционеры** (управление температурой и режимами), **пылесосы** (управление уборкой), **краны** (открытие и закрытие), **лампы** (яркость, цвет, цветовая температура), **рулонные шторы / жалюзи** (открытие/закрытие, позиционирование) и **датчики протечки** (уведомления об обнаружении воды)
- Мгновенно передаёт изменения состояний из HA в Сбер и обратно
- Управляется через удобную панель прямо в интерфейсе Home Assistant

## Требования

- Home Assistant **2026.3** или новее
- Логин и пароль MQTT контроллера от Сбера (получается в Sber Studio — см. ниже)
- Long-Lived Access Token вашего пользователя HA

## Получение логина и пароля MQTT от Сбера

### 1. Регистрация в Sber Studio

Перейдите на портал разработчика: [https://developers.sber.ru/studio/registration](https://developers.sber.ru/studio/registration) и зарегистрируйте учётную запись.

### 2. Создание проекта

После входа нажмите **«Создать проект»**. В открывшемся окне найдите и выберите тип **Smart Home MQTT**. Дайте проекту любое название.

### 3. Создание контроллера

Внутри проекта создайте **контроллер** — это и есть точка подключения к брокеру Сбера. После создания в карточке контроллера будут видны:

- **Логин контроллера** (`sber-mqtt_login`) — используется как логин при подключении к брокеру
- **Пароль контроллера** (`sber-mqtt_password`) — **не** пользовательский пароль от аккаунта, а отдельный пароль контроллера. Если поле пустое — нажмите **«Сгенерировать»** рядом с полем «Пароль контроллера» и сохраните полученное значение.

> Если у вас уже есть плагин SberGate для интеграции HA со Сбером — логин и пароль те же самые, новый проект создавать не нужно.

## Установка через HACS

1. Откройте **HACS → Интеграции → ⋮ (три точки) → Пользовательские репозитории**
2. Вставьте URL репозитория: `https://github.com/TohaRG2/MQTT-Sber-HA`
3. Тип: **Integration** → нажмите **Добавить**
4. Найдите в списке **Sber MQTT Bridge** → нажмите **Скачать**
5. Перезапустите Home Assistant

## Настройка интеграции

Перед настройкой подготовьте **Long-Lived Access Token** — он понадобится на шаге 3:

1. Нажмите на **ваш профиль** в левом нижнем углу HA
2. Прокрутите страницу вниз до раздела **«Долгосрочные токены доступа»**
3. Нажмите **«Создать токен»**, введите любое название (например, `Sber MQTT`)
4. Скопируйте токен — он показывается только один раз

Затем добавьте интеграцию:

1. Перейдите в **Настройки → Устройства и службы → + Добавить интеграцию**
2. Найдите **Sber MQTT Bridge**
3. Введите данные для подключения:
   - **Логин MQTT** — логин контроллера из Sber Studio
   - **Пароль MQTT** — пароль контроллера из Sber Studio
   - **Long-Lived Access Token** — токен, созданный выше
   - **Брокер** и **Порт** — оставьте по умолчанию
4. Нажмите **Отправить** — интеграция проверит соединение и добавится в HA

После успешной настройки в боковом меню появится раздел **Sber MQTT**.

## Добавление устройств

Откройте панель **Sber MQTT** в боковом меню и нажмите **«+ Добавить устройство»**.

### Реле (выключатель, лампа, кнопка, сценарий)

Прокидывает устройства HA в Салют — чтобы их можно было включать и выключать голосом или удалённо через приложение.

Подходит для сущностей с доменами: `switch`, `light`, `input_boolean`, `media_player`, `script`, `button`, `input_button`.

> **Примечание:** `script`, `button`, `input_button` — запускаются по команде от Сбера, собственного состояния on/off не имеют. `media_player` — состояние `off` считается выключенным, все остальные (`on`, `idle`, `playing`, `paused`) — включённым.

Помимо управления включением/выключением, реле поддерживает **энергомониторинг** — передачу текущих показаний мощности, тока и напряжения в Салют. Сенсоры задаются опционально при добавлении устройства:

| Параметр | Единица | Сенсор HA |
|---|---|---|
| Мощность | Вт | `sensor` с `device_class: power` |
| Ток | мА | `sensor` с `device_class: current` |
| Напряжение | В | `sensor` с `device_class: voltage` |

При выборе основной сущности интеграция автоматически подбирает соответствующие сенсоры того же устройства HA. Если автоподбор не сработал — сенсоры можно выбрать вручную или оставить пустыми.

1. Выберите тип **Реле**
2. Найдите и выберите нужную сущность из списка (можно искать по имени или комнате)
3. При необходимости выберите сенсоры энергомониторинга — или оставьте пустыми
4. Задайте имя, ID и комнату — заполняются автоматически из HA, при необходимости отредактируйте
5. Нажмите **«Готово»**


### Сценарная кнопка

Позволяет прокидывать события из HA в Сбер — чтобы в приложении Салют можно было создавать сценарии, реагирующие на изменения состояний устройств HA.

Подходит для тех же доменов что и реле: `switch`, `light`, `input_boolean`, `media_player`, `script`, `button`, `input_button`.

Логика преобразования событий:
- Включение (`on`) → **click** (однократное нажатие)
- Выключение (`off`) → **double_click** (двойное нажатие)
- Нажатие кнопки/запуск сценария (`button`, `input_button`, `script`) → **click**

1. Выберите тип **Сценарная кнопка**
2. Найдите и выберите нужную сущность из списка
3. Задайте имя, ID и комнату
4. Нажмите **«Готово»**

После добавления в приложении Салют появится устройство типа «Сценарная кнопка». В разделе **Сценарии** можно создать автоматизацию с триггером «нажатие» на эту кнопку.

### Датчик температуры/влажности

1. Выберите тип **Датчик температуры/влажности**
2. Укажите сенсоры HA для каждого параметра (все опциональны, но температура или влажность обязательно):
   - 🌡 Температура
   - 💧 Влажность
   - 🔋 Заряд батареи
   - 📶 Сила сигнала
3. Задайте имя, ID и комнату
4. Нажмите **«Готово»**

### Кондиционер

Подходит для сущностей домена `climate`. Позволяет управлять кондиционером голосом через Салют: включать/выключать, задавать целевую температуру и режим работы.

Поддерживаемые режимы работы:

| Режим в HA | Режим в Салюте |
|---|---|
| `cool` | Охлаждение |
| `heat` | Нагрев |
| `fan_only` | Вентиляция |
| `dry` | Осушение |
| `auto` / `heat_cool` | Авто |

1. Выберите тип **Кондиционер**
2. Найдите и выберите нужную `climate`-сущность из списка
3. При желании укажите **датчик текущей температуры** — если не выбрать, используется встроенный датчик кондиционера (`current_temperature` из атрибутов сущности)
4. Задайте имя, ID и комнату
5. Нажмите **«Готово»**

После добавления в Салюте можно голосом говорить: «Салют, включи кондиционер», «Салют, установи температуру 22 градуса», «Салют, включи режим охлаждения на кондиционере».

### Пылесос

Подходит для сущностей домена `vacuum`. Позволяет управлять пылесосом голосом через Салют: запускать уборку, ставить на паузу, отправлять на базу.

Поддерживаемые статусы:

| Статус в HA | Статус в Салюте |
|---|---|
| `cleaning` | Уборка |
| `docked` | На базе |
| `paused` / `idle` / `error` | Пауза |
| `returning` | Возвращение на базу |

1. Выберите тип **Пылесос**
2. Найдите и выберите нужную `vacuum`-сущность из списка
3. Датчик заряда батареи подтягивается автоматически — если HA знает о нём как об отдельной сущности того же устройства. При необходимости можно выбрать вручную или оставить пустым (тогда используется атрибут `battery_level` самого пылесоса)
4. Задайте имя, ID и комнату
5. Нажмите **«Готово»**

После добавления в Салюте можно голосом говорить: «Салют, включи пылесос», «Салют, верни пылесос на базу», «Салют, поставь пылесос на паузу».

### Кран

Прокидывает краны и вентили в Салют — чтобы их можно было открывать и закрывать голосом или через приложение.

Подходит для сущностей доменов `valve` и `switch`. Домен `valve` дополнительно поддерживает команду **stop** (остановить движение), `switch` — только открытие и закрытие.

Поддерживаемые состояния:

| Состояние в HA | Состояние в Салюте |
|---|---|
| `open` / `opening` / `on` | Открыт |
| `closed` / `closing` / `off` | Закрыт |

1. Выберите тип **Кран**
2. Найдите и выберите нужную сущность (`valve` или `switch`) из списка
3. Задайте имя, ID и комнату
4. Нажмите **«Готово»**

После добавления в Салюте можно голосом говорить: «Салют, открой кран», «Салют, закрой воду в кладовой», «Салют, останови кран».

### Лампа

Прокидывает управляемые лампы HA в Салют. Помимо включения и выключения, поддерживает расширенное управление — в зависимости от возможностей конкретной лампы.

Доступные функции (выбираются при добавлении устройства):

| Функция | Описание |
|---|---|
| **Яркость** | Регулировка от 0 до 100% |
| **Цвет** | Управление цветом в формате HSV |
| **Цветовая температура** | От холодного белого до тёплого |
| **Режим** | Переключение между белым и цветным режимом |

При добавлении лампы интеграция автоматически определяет, какие функции поддерживает лампа, и предлагает включить только их. Любую функцию можно отключить, если не нужна в Салюте.

1. Выберите тип **Лампа**
2. Найдите и выберите нужную `light`-сущность из списка
3. Отметьте функции, которые хотите использовать в Салюте — поддерживаемые лампой будут отмечены автоматически
4. Задайте имя, ID и комнату
5. Нажмите **«Готово»**

После добавления в Салюте можно голосом говорить: «Салют, включи лампу», «Салют, сделай свет на 50%», «Салют, включи красный цвет», «Салют, сделай свет теплее».

### Рулонные шторы / жалюзи

Подходит для сущностей домена `cover`. Позволяет открывать, закрывать и устанавливать нужное положение штор голосом через Салют.

Поддерживаемые состояния:

| Состояние в HA | Состояние в Салюте |
|---|---|
| `open` | Открыто |
| `opening` | Открывается |
| `closed` | Закрыто |
| `closing` | Закрывается |

1. Выберите тип **Рулонные шторы / жалюзи**
2. Найдите и выберите нужную `cover`-сущность из списка
3. Датчик заряда батареи подтягивается автоматически — если HA знает о нём как об отдельной сущности того же устройства. При необходимости можно выбрать вручную или оставить пустым
4. Задайте имя, ID и комнату
5. Нажмите **«Готово»**

После добавления в Салюте можно голосом говорить: «Салют, открой шторы», «Салют, закрой жалюзи», «Салют, подними шторы на 50%», «Салют, останови шторы».

### Датчик протечки

Прокидывает датчики протечки HA в Салют. Подходит для сущностей домена `binary_sensor` с `device_class: moisture`.

Когда датчик фиксирует воду (`on`), Салют получает уведомление — его можно использовать в автоматизациях и сценариях приложения.

1. Выберите тип **Датчик протечки**
2. Найдите и выберите нужный `binary_sensor` из списка — отображаются только сущности с `device_class: moisture`
3. Датчик заряда батареи подтягивается автоматически, если HA знает о нём как об отдельной сущности того же устройства
4. Задайте имя, ID и комнату
5. Нажмите **«Готово»**

## Управление устройствами

В панели **Sber MQTT** отображается список всех добавленных устройств с их текущим состоянием.

- **Обновить в Сбере** — принудительно переотправить конфигурацию всех устройств в Сбер. Используйте если устройства пропали из приложения Салют после перезапуска HA.
- **Отправить состояния** — принудительно отправить текущие состояния всех устройств в Сбер. Используйте если состояния в приложении Салют не соответствуют реальным.
- **🗑** — удалить устройство. После удаления оно пропадёт из приложения Сбера.

## Ручная установка (без HACS)

1. Скачайте репозиторий или архив с релизом
2. Скопируйте папку `custom_components/sber_mqtt` в `/config/custom_components/sber_mqtt/`
3. Перезапустите Home Assistant
4. Продолжайте с шага «Настройка интеграции»
